Quantum Annealing¶

  • Not equivalent to universal gate based quantum computing
  • Takes an initial Hamiltonian and slowly evolves it (via adiabatic theorem)
  • Encodes a solution the ground state of the final Hamiltonian
  • Only "stoquastic hamiltonians" (ground state can be described as classical probability)

Hardware¶

  • D-Wave 2000Q (aims for 2000 qubit yield)
  • "Chimera" graph - limited connectivity
  • Superconducting loops used as qubits

What can we do with this?

  • Optimiser, for example: $\min \left(f(\mathbf{x}) = \mathbf{x}^T Q \mathbf{x}\right)$ (QUBO)
  • Machine learning
  • Materials simulation

Optimising Example - "pick one"¶

\begin{equation} f(\mathbf{x}) = \left(\sum_{i=0}^{N} x_i - 1\right)^2 \end{equation}

Which gives...

\begin{equation} f(\mathbf{x}) = -x_0 -x_1 \dots -x_N + 2x_0x_1 + \dots + 2x_Nx_{N-1} +1 \end{equation}

Linear terms are the diagonal of $Q$, quadratic are off-diagonal

D-Wave library requires it's stored in the upper triangle

The Robots¶

Project Rules¶

  • Project is assessing feasibility
  • What would an application look like?
  • Robots don't accelerate
  • All robots are identical
  • No faults or performance degradation

Method¶

Use annealer to pick optimal combinations of predetermined routes
Generalisation of VW/D-Wave traffic problem

Workflow¶

  • Classical route generation (only considering a single robot)
  • Quantum optimiser
  • Classical validation

Constraints¶

  • One route per robot
    • Same as before: $\left(\sum_{i=0}^{N} x_i - 1\right)^2$
  • No collisions
    • $cost(cell,time) = \left(\sum_{q_{ij}\in c_{time}} q_{ij}\right)^2$
  • Shortest route is better (implied by previous)
  • Minimise turns (add a delay when creating route)
  • Other constraints added as needed!

What about large problems?¶

qbsolv¶

"A decomposing solver that finds a minimum value of a large quadratic unconstrained binary optimization (QUBO) problem by splitting it into pieces."

First results¶

  • OK for first attempt
  • There are still collisions
  • We can do better!

Retrying failed attempts¶

  • Retry with more routes for failed robots
  • Fix robots that were OK
  • Collisions are gone
  • Cheaper than rerunning whole problem

Performance¶

Lower is better

Why so slow?¶

Classical overheads:

  • qbsolv's algorithm
  • embedding (big one!)
  • more work needed here

Time stepping¶

Finishing up the touches on time stepping:

  • When a robot completes a job, it is given a new one
  • Repeat annealing, with new routes and constraints

Takeaways¶

  • The D-Wave architecture forced us to think about out problem in a different way.
  • Hybrid algorithms are needed (don't throw away your CPUs)
  • Embedding the problem on the QPU graph structure is expensive
  • Using domain specific knowledge may help with embedding
